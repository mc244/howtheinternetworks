Domain Names and ICANN



A domain name is the address at which Internet users can access your website. An analogy that might be easier to understand is if your website is a house, then the domain name is the address, which is just like how you send letters to a mailbox as depicted. The Internet is a giant network of computers connected to each other, and our computers are all assigned an IP address, which is an accumulation of several numbers that are unique to each computer. If we had to figure out the IP address to visit a website every time, it would be very difficult to remember. That is why domain names were invented. So how does a domain name work? When the domain name is entered into the web browser, it will send a request to a global network of servers that form the domain name system (DNS). It will keep sending it to different servers until the domain name is recognized as a part of the server database or cache. The server will fetch the webpage and send the data back to the browser.

A domain name generally consists of two parts. One is called the top-level domain or TLD. These are the most generic domain extensions such as .com, .org, and .net. The other part is called the subdomain, which is the address before the top-level domain. For example, in google.com, the subdomain is google, while the top-level domain is .com.

The Internet Corporation for Assigned Names and Numbers (ICANN) is the organization responsible for managing the domain names system. It is a non-profit organization that creates and implements the policies for domain names. ICANN is also responsible for coordinating the management of the technical elements of the DNS to ensure that all Internet users can find valid addresses. While the organization is responsible for the domain name itself, is not responsible for any activities such as financial transactions or content control performed in the domain itself.
